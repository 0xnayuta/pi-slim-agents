# AGENTS.md

## 项目概述

`pi-slim-agents` 是一个轻量级 `pi-mono` 专业代理扩展，为开发工作流提供小巧、可定制的代理委托层。专注于提示配置、命令工程、验证、模板、历史、指标和 JSON 输出，有意避免繁重的子代理编排。

## 核心原则

- 保持轻量、透明、易懂。
- 优先简单明确的行为，而非隐藏的自动化。
- 除非明确请求，不引入繁重编排。
- 保持代理角色界限清晰，避免职责重叠。
- 基于 Markdown 的易读代理定义。
- 不误导用户：不声称未执行的操作，坦诚说明限制。

## 非目标

除非明确请求并经评估，不添加：子代理编排框架、后台会话、子会话委托、工作树隔离、调度器、MCP 集成、多代理投票、TUI 面板管理、无实际提供商支持的跟踪、基于不稳定 API 的集成。

## 代码库结构

- `src/` — TypeScript 源码
- `agents/` — 内置代理提示配置
- `templates/` — 可选代理模板
- `skills/` — 技能定义
- `docs/` — 文档
- `examples/` — 示例与评估材料
- `scripts/` — 验证与打包脚本
- `tests/` — 自动化测试

除非明确请求，不重命名公共目录或文件。

## 开发规则

**修改前：** 先阅读相关文件，确定最小安全更改，简要说明预期改动。

**修改时：**

- 保持更改聚焦，避免大规模重构。
- 优先共享辅助函数，避免重复逻辑。
- 保持文本和 JSON 输出行为一致，JSON 必须有效且含稳定字段（如 `schemaVersion`）。
- 尽量保持模式稳定。
- 不在输出中暴露绝对路径、API 密钥、堆栈跟踪、完整提示或敏感环境变量值。显示错误前先清理。

**代理和模板：**

- 使用带 frontmatter 的 Markdown 格式。
- 明确说明代理应做和不应做的。
- 保持提示简洁，避免职责重叠。
- 只读代理不声明修改文件；可写代理不擅自扩展权限。
- 保留有效 frontmatter 字段，不翻译或重命名配置字段。

**文档：** 行为变化时更新文档，保持准确、简洁、实用、以示例为导向，不将实验功能描述为稳定功能。

**测试：** 更改后运行 `pnpm typecheck && pnpm build && pnpm test`，覆盖正常路径、错误路径、边界情况、JSON 输出和隐私清理行为。不依赖网络访问或私有本地状态。

## 安全与隐私

不暴露：API 密钥、npm 令牌、授权标头、环境变量值、完整堆栈跟踪、绝对路径、完整提示或完整代理结果。使用安全显示路径，错误信息展示前应清理。
